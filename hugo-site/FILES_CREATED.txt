================================================================================
MIGRATION SCRIPTS
================================================================================

12. migrate-blogs.py
    - Python 3 script for bulk HTML → Markdown conversion
    - Parses HTML blog files using html.parser
    - Extracts: title, date, description, category, subtitle, read time
    - Extracts related posts from ba-related-grid div
    - Converts HTML body to Markdown (with raw HTML preserved)
    - Usage: `python3 migrate-blogs.py --source ../blog --dest ./content/blog`
    - Processes 68+ blog posts automatically
    - Detailed error reporting and progress tracking
    - ~350 lines

13. migrate-blogs.sh
    - Bash alternative to Python migration script
    - Uses grep/sed for HTML extraction
    - Similar functionality to Python version
    - Less robust but good fallback on systems without Python 3
    - Usage: `./migrate-blogs.sh ../blog`
    - ~200 lines

NEXT STEPS FOR USER
================================================================================

PHASE 1: Testing (5-10 minutes)
  1. Install Hugo: `brew install hugo` (or platform equivalent)
  2. Navigate: `cd hugo-site`
  3. Test: `hugo server -D`
  4. Visit: http://localhost:1313
  5. Create test post: `hugo new blog/test.md`
  6. Verify everything renders correctly

PHASE 2: Content Migration (30 minutes to 2 hours)
  1. Run migration script: `python3 migrate-blogs.py --source ../blog`
  2. Review generated content/blog/ files
  3. Manually edit any posts that need adjustment
  4. Verify all front matter is correct
  5. Check for any rendering issues

PHASE 3: Deployment Prep (30 minutes)
  1. Review DEPLOYMENT.md
  2. Create Git repository and push code
  3. Connect to Netlify or deployment platform
  4. Run pre-deployment checklist
  5. Deploy!

PHASE 4: Go Live
  1. Deploy to production
  2. Verify site works at custom domain
  3. Test all functionality
  4. Monitor build logs and performance
  5. Set up SSL/HTTPS (usually automatic)
