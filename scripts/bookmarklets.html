Manual Price Scrape — Bookmarklets (v3)

Two paste-based bookmarklets. Drag each to your bookmarks bar once, then reuse weekly. Each prompts you to paste its targets JSON, so the bookmarklet itself doesn't need editing when target lists change.

Why this exists: Cost Plus Drugs and HealthWarehouse both have anti-scraping measures (Cloudflare 403, Next.js soft-404 catchall) that block plain Node fetches. Running these from a real Chrome tab on the source domain bypasses both, because your browser session has the necessary cookies. As of April 2026, CPD also moved to client-side rendering — its data now streams as escaped JSON inside __next_f.push(...) blocks, so the v3 extractor looks for priceCalculation anchored to the URL's slug instead of the legacy JSON-LD product schema.

Weekly workflow

Phase 1 — generate target lists

cd ~/Documents/html/psychopharm
node scripts/scrape-prices.js

This populates NADAC into data/prices.json and writes data/cpd-targets.json and data/hw-targets.json.

Phase 2 — Cost Plus Drugs scrape

  1. Copy CPD targets to clipboard: 
    cat data/cpd-targets.json | jq .targets | pbcopy
  2. In Chrome, navigate to a real CPD product page (not the homepage, not a category listing). Easiest: open https://www.costplusdrugs.com/medications/fluoxetine-20mg-capsule/ and wait until the price shows. This warms Cloudflare and the React app.
  3. Click the Scrape CPD bookmarklet. When prompted, paste targets and click OK.
  4. After ~10 seconds an alert reads "CPD scrape v3: X/Y prices captured." Healthy result is X ≥ 40 (out of ~47). If X is much lower, see Troubleshooting before continuing.
  5. Move the downloaded file: 
    mv ~/Downloads/cpd-prices.json ~/Documents/html/psychopharm/data/

Phase 3 — HealthWarehouse scrape

  1. Copy HW targets to clipboard (note: different file from Phase 2): 
    cat data/hw-targets.json | jq .targets | pbcopy
  2. Navigate to a real HW product page in the same window, e.g. https://www.healthwarehouse.com/sertraline-50mg-tablets-generic-zoloft. Wait until the page renders.
  3. Click the Scrape HW bookmarklet. Paste targets, OK.
  4. After ~10 seconds the alert reports the count. Move it: 
    mv ~/Downloads/hw-prices.json ~/Documents/html/psychopharm/data/
Don't cross-paste targets. The CPD targets file maps drug names to single URL strings; HW maps drug names to arrays of candidate URLs. Always re-run pbcopy with the right file before each bookmarklet. The v3 bookmarklets detect mismatched input and refuse to run with a clear error message — but it's wasted time.

Phase 4 — verify and merge

  1. Sanity-check both inputs are present and non-trivial: 
    ls -lh data/cpd-prices.json data/hw-prices.json
jq ".results | to_entries | map(select(.value.available)) | length" \
  data/cpd-prices.json data/hw-prices.json
    Each file should be a few KB; counts should match what each bookmarklet's alert reported.
  2. Merge into the main prices file (mirrors automatically to hugo-site/static/data/): 
    node scripts/merge-retail-prices.js
  3. Commit: 
    git add data/ hugo-site/static/data/
git -c user.name="Jerad Shoemaker" -c user.email="jshoemakerhwc@gmail.com" \
    commit -m "Update drug prices $(date +%F)"
git push

The bookmarklets

Drag each link to your bookmarks bar.

If you used previous (v1 or v2) versions, delete those bookmarks first. They use obsolete JSON-LD or naive priceCalculation logic and will return 0/X on the current CPD site. Replace with these v3 versions.

Drag-to-bookmarks not working? Some Chrome flags block drag-as-bookmark from inline JavaScript URLs. Two workarounds:
  1. Right-click an existing bookmark, choose "Add page". Right-click the green button above and "Copy link address" to grab its URL, paste into the URL field, name it Scrape CPD or Scrape HW.
  2. Console paste route: open Chrome DevTools (Cmd+Opt+I) on the source site, click the Console tab. If Chrome blocks pasting, type literally allow pasting and press Enter. Then right-click the bookmarklet button above, "Copy link address", paste into the Console (the javascript: prefix is harmless), press Enter.

Pre-flight checks

Before clicking each bookmarklet, confirm:

  • You're on a product page on the matching domain (URL contains a specific drug name, not just /medications/ or /categories/).
  • Real prices/data are visible on the page (proves Cloudflare cleared and JS rendered).
  • You just pbcopy'd the matching *-targets.json.
  • If a 0/X result comes back, do not proceed to merge — diagnose first.

What each bookmarklet does

Scrape CPD (v3)

  1. Validates pasted input is CPD-shape (string-valued targets). Auto-unwraps if you pasted the full file with a generatedAt wrapper.
  2. For each {drugName: productUrl} pair, fetches the product page in same-origin context with credentials.
  3. CPD's HTML now streams data inside __next_f.push(...) calls as escaped JSON. The v3 extractor:
    • Pulls the slug from the URL (e.g. fluoxetine-20mg-capsule).
    • Finds priceCalculation followed within ~500 chars by slug:"<that-slug>".
    • Slug anchoring is critical: each product page contains prices for all strength variants (10mg, 20mg, 60mg, etc.). Naive matching grabs the first variant, not the canonical one.
  4. Concurrency 6 — 47 drugs finish in seconds.

Scrape HW (v3)

  1. Validates pasted input is HW-shape (array-valued candidate lists per drug).
  2. For each {drugName: [url1, url2, ...]} pair, tries URLs in order until one succeeds.
  3. For each successful response, tries three extraction strategies in order, recording which one matched:
    1. JSON-LD offers[] — legacy HW format with pack-size eligibleQuantity; picks the offer closest to a 30-pill pack.
    2. Slug-anchored priceCalculation — same as CPD v3, in case HW migrates to Next.js.
    3. Slug-anchored generic price field — broader fallback.
  4. Each successful entry includes a strategy field so you can tell at a glance which path worked.

Troubleshooting

SymptomCause / fix
CPD: HTTP 403 on every drugCloudflare doesn't recognize your session. Visit a real product page first, click around to a second page, then re-run.
CPD: no priceCalculation found on every drugCPD changed page structure again. Use the diagnostic below to save sample HTML, identify the new field name, update the regex in this file.
HW: HTTP 429 on every drugRate-limited (often after a cross-paste blast). Wait 15–30 minutes, then retry with the correct HW targets.
HW: all candidates failed with errors mentioning (/) as the URLYou pasted CPD targets (string values) into the HW prompt and the bookmarklet character-iterated them as bogus URLs. v3 detects this up-front; if you see this error, you're on an old bookmarklet — re-install v3.
Empty result file / no download triggeredBookmarklet errored before download. The alert dialog has the message; usually a JSON paste typo or a CSP block.
Result count is suspiciously low (e.g., 47 entries when expecting 61)Wrong targets pasted. Re-copy the matching *-targets.json and re-run.
Merge says both retail sources returned zero pricesMerge refuses to overwrite good data with zero data. This is a guard, not a bug. Diagnose which source failed and re-run before re-merging.
Lost data/prices.json or data/drug-skus.jsonThey're tracked in git. Restore: git checkout HEAD -- data/prices.json data/drug-skus.json. Then regenerate targets via Phase 1.

Diagnostic — when a site changes structure

If a future site change breaks v3, get a sample of the actual HTML the bookmarklet sees, then update the regex. With allow pasting enabled in the Console, paste this on a CPD or HW product page:

(async()=>{
  const url = location.href;
  const r = await fetch(url, {credentials:"include"});
  const h = await r.text();
  const blob = new Blob([h], {type:"text/html"});
  const a = document.createElement("a");
  a.href = URL.createObjectURL(blob);
  a.download = "page-sample.html";
  document.body.appendChild(a); a.click(); a.remove();
  console.log("HTTP", r.status, "length", h.length,
              "has priceCalculation:", h.includes("priceCalculation"));
})();

Save the file to data/page-sample.html and grep for the visible price; the surrounding text shows the new field name. Update the regex in the bookmarklet href above accordingly.
